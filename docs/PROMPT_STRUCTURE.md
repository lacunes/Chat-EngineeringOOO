# Prompt 结构说明

## 当前结构（四层）

```
┌─────────────────────────────────────────┐
│ msg 0: [固定层]                         │  ← DeepSeek prefix cache 从此处开始匹配
│   world.SYSTEM_PROMPT                   │
│   + TIME_INJECT_INSTRUCTION             │
├─────────────────────────────────────────┤
│ msg 1: [半固定层]                       │
│   [长期记忆]                            │
│   (仅精炼/新增/删除时变化)              │
├─────────────────────────────────────────┤
│ msg 2: [动态层]                         │
│   NPC 舞台指令                          │
│   + 关系摘要                            │
│   + 时间数据                            │
│   + 剧情状态                            │
│   + runtime_directive                   │
├─────────────────────────────────────────┤
│ msg 3+: [对话层]                        │
│   最近 N 条聊天记录                     │
├─────────────────────────────────────────┤
│ [当前输入]                              │
│   用户消息（在最末条消息中）             │
└─────────────────────────────────────────┘
```

## 为什么固定内容放最前面

DeepSeek API 的 prefix cache 从前缀开始匹配。如果 prompt 开头是固定不变的内容，那么这段内容的 KV cache 可以被完全复用，节省推理时间和费用。

- `world.SYSTEM_PROMPT` 在同一个世界中永远不变
- `TIME_INJECT_INSTRUCTION` 是纯指令模板，不含具体时间值
- 这两段合在一起 → **100% cache 命中**

## 为什么动态内容放后面

- NPC 舞台指令：每轮可能触发不同的 NPC
- 关系摘要：数值每轮可能变化
- 时间数据：时段/天数变化
- 剧情状态：手动编辑频率高
- runtime_directive：随时可能开关

这些内容每轮都可能变化，如果放在固定内容前面，会导致整个 prompt 的 cache 全部失效。

## 为什么不能为了缓存删除剧情信息

缓存命中率是成本优化手段，**聊天体验是第一目标**。

- 删除动态信息 = 模型看不到 NPC 行动、关系状态、时间背景
- 没有时间背景 → 模型不知道现在是白天还是夜晚
- 没有关系状态 → NPC 对话语气失去依据
- 没有 NPC 指令 → 世界失去活力

**宁可 cache miss，不能剧情 miss。**

## 哪些内容属于动态层（不应进入固定层）

- ❌ NPC 舞台指令
- ❌ 关系摘要（`[当前角色关系]`）
- ❌ 具体时间数据（`第3天，夏，下午`）
- ❌ 剧情状态（`[当前剧情状态]`）
- ❌ runtime_directive 导演指令
- ❌ 任何含具体数值的动态信息

## 哪些内容属于固定层（可以进入 msg 0）

- ✅ world.SYSTEM_PROMPT（世界设定、写作规则）
- ✅ TIME_INJECT_INSTRUCTION（时段→描写的映射指令模板）
- ✅ RELATION_INJECT_INSTRUCTION（关系数值→行为倾向的指令模板）
- ✅ NPC_STAGE_DIRECTION_INSTRUCTION（舞台指令→叙事融合的指令模板）

注意：指令模板是固定的，具体数据是动态的。例如 `TIME_INJECT_INSTRUCTION` 说"当看到 [当前时间] 段落时请根据时段调整描写"，这是指令模板；实际的时间数据 "第3天 夏 下午" 是动态数据，在 msg 2 中。

## 长期记忆的位置

长期记忆放在 msg 1（半固定层），介于固定和动态之间：

- 它比世界观更易变（精炼/新增/删除时会变化）
- 但它比 NPC/关系/时间更稳定（通常数轮不变）
- 放在固定层之后 → 大部分情况下也能命中 cache
