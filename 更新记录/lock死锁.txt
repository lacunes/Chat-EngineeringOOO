重点检查 bot/llm_router.py 的锁使用。

当前 _state_lock 使用 threading.Lock，但 _update_provider_state() 在持有 _state_lock 时调用 _save_state()，而 _save_state() 内部再次 with _state_lock，导致不可重入锁死锁。

请修复：
1. 将 _state_lock 改为 threading.RLock()，或者重构 _save_state()，避免持锁状态下再次获取同一把锁。
2. 确保 _record_success、_record_failure、_mark_exhausted、_clear_exhausted 不会死锁。
3. 给 LLMRouter.chat() 增加关键路径日志：
   - 收到请求
   - 选中的 provider
   - API 请求开始
   - API 请求结束
   - 开始记录 provider state
   - state 保存完成
   - 准备返回 reply
4. 修复 httpx 日志泄露 Telegram Bot Token 的问题，将 httpx/httpcore 日志级别设为 WARNING。
5. 不要先大改其它功能，先修复 Bot 卡死问题。
