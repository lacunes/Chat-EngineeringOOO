请阅读当前项目代码，对 DeepSeek 聊天请求的 prompt 组装逻辑做一次“缓存友好但不牺牲聊天体验”的重构。

项目目标：
这是一个 Telegram AI Roleplay Framework，核心体验是长期角色扮演、长期记忆、短期上下文、NPC 主动行为、动态关系和世界观切换。不要为了提升缓存命中率而删除、弱化或跳过任何剧情信息。

本次修改目标：
优化 DeepSeek API 的 prefix cache 命中率。原则是：
1. 固定内容尽量放在 prompt 最前面。
2. 动态内容尽量放在固定内容之后。
3. 不减少模型看到的信息。
4. 不减少最近聊天记录。
5. 不关闭长期记忆。
6. 不关闭 NPC 主动行为。
7. 不改变角色回复风格。
8. 不改数据库结构，除非确实必要。
9. 修改后必须保持现有功能可运行。

请重点查找以下文件或类似功能位置：
- deepseek_client.py
- telegram_handlers.py
- memory_manager.py
- npc_manager.py
- worlds/*.py
- 任何负责构造 messages / system prompt / user prompt 的代码

我希望你完成以下改动：

一、拆分 prompt 结构

请把发送给 DeepSeek 的 messages / prompt 组织为更稳定的结构：

推荐顺序：

[固定层 1：世界观固定设定]
- world.SYSTEM_PROMPT 或当前世界的基础设定
- 角色身份设定
- 固定写作规则
- 固定安全边界 / 输出规则
- 不随每轮对话变化的系统规则

[固定层 2：相对稳定的长期记忆]
- 用户长期记忆
- 角色长期记忆
- 世界长期记忆
- 已沉淀的重要事实
注意：不要每轮重排长期记忆。保持顺序稳定。

[动态层：当前状态]
- 当前时间
- 当前地点
- 当前剧情状态
- 当前关系变化
- NPC 舞台指令
- runtime_directive
- 临时导演指令
- 本轮需要特别注意的信息

[短期上下文层]
- 最近 N 条聊天记录，保持原有数量和逻辑，不要减少

[当前输入]
- 当前用户消息

关键要求：
不是删除动态信息，而是把动态信息从 system prompt 最前面挪到固定设定之后。
如果目前 NPC 舞台指令、时间、关系摘要、runtime_directive 被拼接在 system prompt 开头，请调整为放在固定设定之后。

二、保持体验不变

不要做以下事情：
- 不要减少最近聊天记录数量
- 不要减少长期记忆内容
- 不要关闭记忆提取
- 不要关闭摘要压缩
- 不要关闭 NPC 主动行为
- 不要把动态关系、当前状态、时间、NPC 指令移除
- 不要把角色回复变短
- 不要修改角色人设文本内容，除非只是移动位置
- 不要为了缓存强行固定剧情状态

三、增加 cache usage 日志

请检查 DeepSeek API 返回的 usage 字段，尽量记录以下信息：

- prompt_cache_hit_tokens
- prompt_cache_miss_tokens
- prompt_tokens
- completion_tokens
- total_tokens
- 当前请求类型，例如：
  - main_chat
  - memory_extract
  - memory_compress
  - relation_extract
  - continue_reply
  - npc_action
  - unknown

如果已有日志系统，请接入现有日志。
如果没有，请用项目已有 logger 或 print 做清晰输出。

日志示例：

[DeepSeek Usage][main_chat] hit=1234 miss=5678 prompt=6912 completion=800 total=7712 hit_rate=17.85%

注意：
hit_rate = prompt_cache_hit_tokens / (prompt_cache_hit_tokens + prompt_cache_miss_tokens)
如果字段不存在，不要报错，记录为 0 或 N/A。

四、尽量封装 prompt builder

如果当前 prompt 组装逻辑比较分散，请尽量抽出一个清晰函数，例如：

build_chat_messages(
    world_prompt,
    stable_rules,
    long_term_memory,
    dynamic_state,
    recent_messages,
    current_user_message

或者在现有结构中拆成类似函数：

build_stable_system_prompt()
build_dynamic_state_prompt()
build_recent_context_messages()

但不要大规模重构，不要引入复杂架构。以低风险修改为主。

最终目标：
在不牺牲角色扮演体验的前提下，让 DeepSeek prefix cache 更容易命中。优先保证聊天质量，其次才是缓存命中率。
