当前问题是 Web 世界编辑器直接解析和改写 Python 源码，导致容易露出源码、字段解析失败、保存失败。

目标：
把 worlds 的 Web 编辑从“直接编辑 .py 源码”改为“编辑结构化数据文件”。

要求：
1. 保留现有 Python 主项目。
2. 不要重写整个项目。
3. 不要改变聊天逻辑。
4. 不要改变 prompt 结构。
5. 不要增加 API 调用。
6. 不要修改现有世界文本内容。
7. 优先低风险迁移。

建议方案：
1. 新增 data/worlds/ 目录。
2. 为每个世界创建 JSON 数据文件，例如 data/worlds/one.json。
3. JSON 字段包括：
   - world_name
   - start_scene
   - system_prompt
   - characters
   - rules
   - locations
   - event_pool
   - npcs
4. 新增 world_loader.py，负责从 JSON 加载世界数据。
5. Web 世界编辑器只读取和保存 JSON，不再直接解析/改写 bot/worlds/*.py。
6. 保存前做字段校验。
7. 保存时使用原子写入：
   - 先写临时文件
   - 校验 JSON
   - 再替换正式文件
8. 保存前自动备份旧 JSON 到 backups/worlds/
9. 如果 JSON 不存在，允许从旧的 bot/worlds/*.py 读取一次并迁移。
10. 旧的 bot/worlds/*.py 暂时保留作为兼容，不要删除。

重点修复：
- Web 面板不再暴露 Python 源码。
- Web 保存失败时不能破坏原世界文件。
- CHARACTERS / RULES / LOCATIONS / EVENT_POOL / NPCS 都应稳定显示和保存。
- 多行文本要稳定保存。
- 保存失败要给出明确错误信息。

1. 修改了哪些文件
2. 新的世界数据保存在哪里
3. Web 是否还会直接编辑 .py
4. 是否兼容旧 worlds/*.py
5. 是否修改聊天逻辑
6. 是否增加 API 调用
