请阅读当前项目，重点分析并小幅优化长期记忆系统。

项目背景：
这是一个 Telegram AI Roleplay Framework，核心体验是长期角色扮演。项目已有短期上下文、长期记忆、摘要压缩、NPC 主动行为、动态关系、世界观切换等功能。

本次目标：
优化“长期记忆质量”，不是增加记忆数量，也不是增加每轮 API 调用。

非常重要：
1. 不要增加每轮主聊天的 API 调用次数。
2. 不要让每轮聊天都额外调用记忆提取/记忆审核。
3. 不要减少短期上下文。
4. 不要删除已有长期记忆功能。
5. 不要关闭摘要压缩。
6. 不要影响 NPC 主动行为。
7. 不要改角色人设文本。
8. 不要大规模重构。

请重点查看：
- bot/memory_manager.py
- bot/telegram_handlers.py
- bot/deepseek_client.py
- bot/settings.py
- bot/worlds/*.py
- 任何和 long_memory、memory_extract、summary、compress 相关的代码

我希望你先分析当前记忆系统：
1. 长期记忆在哪里生成？
2. 长期记忆在哪里保存？
3. 长期记忆什么时候触发？
4. 长期记忆是否会每轮调用 API？
5. 长期记忆是否有分类？
6. 是否存在记忆污染风险，例如临时情绪被永久记忆、剧情猜测被当成事实、角色误解被写入长期记忆？

然后进行低风险优化：

一、给长期记忆增加轻量分类

如果当前长期记忆只是字符串列表，请尽量在不破坏兼容性的前提下增加分类概念。

建议分类：
- hard_fact：硬设定，不应轻易改变
- relationship：人物关系变化
- plot_fact：已发生剧情事实
- user_preference：用户偏好
- temporary_state：临时状态，后续可过期
- character_state：角色当前状态
- world_state：世界状态

注意：
如果数据库/文件结构改动风险较大，可以先不改底层存储，只在记忆文本前加标签，例如：

[plot_fact] 林栖在雨夜来到陈平家门口。
[relationship] 林栖对陈平的依赖感上升。
[temporary_state] 当前场景在陈平家中。

二、优化记忆提取 prompt

请修改记忆提取的提示词，让模型只提取“值得长期保留”的信息。

要求记忆提取时遵守：
1. 不记录普通寒暄。
2. 不记录短暂情绪，除非它改变了关系或剧情。
3. 不记录模型自己的猜测。
4. 不记录尚未发生的计划为事实。
5. 不把一次性的场景动作当作永久设定。
6. 不覆盖硬设定。
7. 关系变化要谨慎，必须有明确依据。
8. 临时状态要标记为 temporary_state。

三、不要提高 API 调用频率

请确认：
- 记忆提取仍然沿用原有触发频率。
- 不要新增“每轮记忆审核 API”。
- 如果需要审核功能，做成手动触发或低频触发。

四、增加日志

请在记忆提取成功时记录：
- 本次提取了几条记忆
- 每条记忆的分类
- 是否跳过了无价值记忆
- 是否有记忆被判定为 temporary_state

日志不要包含 API key。

五、兼容旧记忆

如果已有旧记忆没有分类，不要报错。
可以默认当作 plot_fact 或 legacy。
不要清空旧记忆。

六、测试

修改后请运行基础语法检查，例如：

python -m py_compile bot/memory_manager.py
python -m py_compile bot/telegram_handlers.py

如果修改了其他 Python 文件，也检查对应文件。

最后请输出：
1. 修改了哪些文件
2. 当前记忆触发频率是否改变
3. 是否增加了每轮 API 调用
4. 新增了哪些记忆分类
5. 旧记忆如何兼容
6. 有哪些风险
7. 如何查看记忆提取日志

核心原则：
提升记忆质量，不提升 API 调用频率。
聊天体验第一，成本控制第二，功能扩展第三。
