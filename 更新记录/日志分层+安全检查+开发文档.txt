二、敏感信息安全检查

请新增一个轻量脚本：

scripts/security_check.sh

功能：
1. 检查 .env 是否被 git 跟踪。
2. 检查 secrets.json 是否被 git 跟踪。
3. 检查 logs/ 是否被 git 跟踪。
4. 检查 backups/ 是否被 git 跟踪。
5. 检查 .gitignore 是否包含以下规则：
   - .env
   - *.env
   - secrets.json
   - logs/
   - backups/
   - __pycache__/
   - *.pyc
6. 扫描项目中的疑似敏感信息。

建议扫描关键词：
- DEEPSEEK_API_KEY
- TELEGRAM_BOT_TOKEN
- BOT_TOKEN
- API_KEY
- Authorization: Bearer
- sk-
- token=
- api_key=
- secret=

要求：
1. 如果发现疑似敏感信息，只输出文件路径和行号。
2. 不要自动删除任何内容。
3. 不要自动改写用户文件，除非只是向 .gitignore 追加缺失规则。
4. 如果要追加 .gitignore，请先保留原内容，只在末尾追加缺失项。
5. 不要扫描 .git 目录。
6. 不要扫描 venv、.venv、node_modules、__pycache__。
7. 不要把真实密钥打印到终端，只打印“疑似敏感内容所在位置”。
8. 脚本应有清晰中文提示。
9. 脚本运行失败不能影响项目代码。
