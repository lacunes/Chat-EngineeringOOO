
我的目标是在这个 Telegram AI Roleplay Framework 里新增一个“关系网络系统”。

项目背景：
这是一个 Telegram + DeepSeek 的长期记忆角色扮演框架。当前已有多世界系统、短期记忆、长期记忆、自动记忆提取、历史摘要压缩、NPC 主动行为、Web 管理面板。我要新增的关系网络系统不是替代长期记忆，而是把角色之间的关系做成结构化数据，让剧情长期推进时更稳定、更有游戏感。

请你先做第一阶段：设计并实现一个最小可用版，不要一次性做太复杂。

功能目标：

1. 每个世界拥有独立的关系网络文件。
   例如：
   memory/one_relationships.json
   memory/two_relationships.json
   memory/three_relationships.json

2. 关系网络记录“角色 A 对角色 B”的关系状态。
   推荐字段：
   - affection：好感，0-100
   - trust：信任，0-100
   - fear：畏惧，0-100
   - dependence：依赖，0-100
   - suspicion：怀疑，0-100
   - hostility：敌意，0-100
   - notes：文字备注列表
   - last_updated：最后更新时间或轮次

3. 支持非对称关系。
   例如：
  “ 陈平 -> 林栖 的 trust” 和 “林栖 -> 陈平 的 trust” 可以不同。

4. 在每2次 AI 回复后，自动根据本轮对话抽取关系变化。
   不要让它每轮都大幅变化，要克制。
   单次变化建议限制在 -3 到 +3，但并非一成不变，如果出现非常重大事件，可以一次性大幅变化。
   如果没有明显变化，就不更新。若大幅变化则需要在AI的回复后面加入提示。如：“AI的回复内容”结束后加上“（“某数值”+ or - xx）”

5. 关系变化应通过现有 DeepSeek API 实现。
   也就是说，在主回复生成后，再调用一次轻量关系抽取 prompt。
   抽取结果必须是 JSON。
   如果 JSON 解析失败，不能影响主聊天流程，只记录日志。

6. 在生成主回复前，把当前关系网络摘要注入上下文。
   但不要注入整个大 JSON，只注入当前世界的简短摘要。
   例如：
   当前重要关系：
   - 林栖 -> 陈平：信任 68，依赖 42，怀疑 15。
   - 陈平 -> 林栖：忠诚 80，怀疑 20。

7. 给 Telegram 增加命令：
   /relations
   显示当前世界主要角色关系摘要，以方便我查询当下各角色关系。

   /relation_full
   显示更完整的关系网络，但注意 Telegram 消息长度，过长要分段。

8. Web 管理面板也尽量加一个“关系网络”页面。
   尝试做在线编辑。
   如果改 Web 面板风险太大，可以先只做 Telegram 命令和 JSON 存储。

9. 系统要和多世界兼容。
   ACTIVE_WORLD=one 时读写 memory/one_relationships.json。
   切换到 two 时读写 memory/two_relationships.json。
   不要把不同世界关系混在一起。

10. 要考虑文件不存在时自动初始化。
    不能因为没有 relationships.json 导致程序崩溃。

11. 要考虑并发或写入损坏风险。
    写 JSON 时尽量使用临时文件替换，避免写到一半损坏。

12. 不要破坏现有功能：
    - Telegram 私聊
    - 长期记忆
    - 自动摘要
    - NPC 主动行为
    - Web 面板
    - VPS 部署方式
